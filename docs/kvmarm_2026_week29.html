[PATCH v15 00/37] arm64: Support for Arm CCA in KVM

58 messages 3 participants Started: 2026-07-15
本邮件线程讨论了针对 KVM 中 ARM CCA 支持的补丁系列（PATCH v15 00/37），主要集中在如何在 KVM 中实现 ARM 的机密计算架构（CCA）支持。

1. **原始补丁内容**：该补丁系列旨在为 KVM 提供对 ARM CCA 的支持，允许在受保护的虚拟机中运行。补丁分为多个部分，涵盖了从基本架构到具体实现的各个方面。

2. **历史讨论要点**：之前的讨论主要集中在如何分离和重构补丁，以便于审查和集成。补丁的设计考虑到了 RMM v2.0 的新特性，并对现有的 RMI 操作进行了调整，以支持状态感知的 RMI 操作（SRO）。

3. **本周的新讨论和进展**：
- **补丁更新**：补丁中增加了对 Realm 创建、内存映射、PSCI 请求处理等功能的支持。特别是，补丁允许用户空间通过 KVM_SET_ONE_REG 接口配置 Realm 的寄存器，并通过 KVM_GET_REG_LIST 获取寄存器列表。
- **功能增强**：引入了对 SVE（Scalable Vector Extension）的支持，允许用户查询和设置 SVE 向量长度。此外，补丁还处理了 Realm 的 MMIO（内存映射输入输出）仿真，确保在 Realm 中的 MMIO 操作能够正确处理。
- **错误处理**：针对未定义异常的处理进行了讨论，强调了 RMM 不允许在 Realm 中注入未定义异常的必要性。
- **最终启用**：在所有相关补丁到位后，确认了 Realm 的支持已经启用，并在 RMM 检测到时激活。

总的来说，本周的讨论和补丁更新进一步完善了 KVM 对 ARM CCA 的支持，确保了在虚拟化环境中能够安全地运行受保护的虚拟机。

[PATCH v2 00/28] KVM: arm64: Add support for FEAT_NV2p1 and FEAT_NV3

57 messages 6 participants Started: 2026-07-14
本邮件线程讨论了针对 KVM 的 ARM64 架构新增的两个特性：FEAT_NV2p1 和 FEAT_NV3 的支持。这些特性旨在改善嵌套虚拟化的性能和功能。

**原始 patch/问题内容**：
Marc Zyngier 提出了一个包含 28 个补丁的系列，主要是为 ARM64 架构的 KVM 添加对 FEAT_NV2p1 和 FEAT_NV3 的支持。FEAT_NV2p1 主要是修复在 NV 配置下从 EL1 访问某些寄存器时缺失的状态位，而 FEAT_NV3 则改变了在 NV 环境中 ERET 的行为，允许更高效的指令执行。

**之前讨论要点**：
在之前的讨论中，Marc 提到这些补丁经过 FVP 模型测试，显示出 L1 客户端的指令减少约 1.5%。此外，针对 FEAT_NV3 的近似仿真在 L2 和 L3 客户端中显示出显著的性能提升。

**本周的新讨论、进展或结论**：
本周的讨论集中在补丁的具体实现和潜在问题上。Marc 针对 NVHCR_EL2 的处理、寄存器的分类以及对 CPTR_EL2 和 CNTHCTL_EL2 的处理进行了详细的补丁更新。多个补丁得到了参与者的审核和反馈，指出了一些潜在的高优先级问题，如寄存器状态泄露和缺乏必要的同步机制。最终，Marc 强调了在实现中需要确保对新特性的正确检测和处理，以避免在嵌套虚拟化中出现问题。整体来看，讨论推动了对新特性的实现进展，确保了代码的健壮性和性能优化。
