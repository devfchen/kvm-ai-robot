[PATCH v5 00/18] kvm/arm: Introduce a customizable aarch64 KVM host model

53 messages 3 participants Started: 2026-05-19
本邮件讨论的主题是关于在 KVM/ARM 中引入可定制的 AArch64 主机模型的补丁系列（PATCH v5 00/18）。该补丁旨在增强当前的 KVM 主机模型，使其能够设置可写的 ID 寄存器字段，从而在不同硬件之间迁移虚拟机时提供更大的灵活性。

在历史讨论中，Eric Auger 提出了补丁的背景，强调了自 v6.7 内核以来，KVM/ARM 允许用户空间覆盖部分 ID 寄存器的值，并且可写字段的列表正在不断增加。补丁系列中包含多个具体实现，例如引入脚本以自动生成系统寄存器属性定义、初始化可写 ID 寄存器等。

在本周的新讨论中，参与者主要对补丁进行了审查，Sebastian Ott 和 Khushit Shah 对多个补丁给予了“Reviewed-by”的反馈，表示代码已准备好合并。同时，Khushit Shah 提出了一些建议，强调在生成的 cpu-idregs.h.inc 中应包含 RES0/1 字段的信息，以确保用户设置的值在架构上是有效的。此外，他还指出了在不同配置下可能出现的问题，尤其是在嵌套虚拟化支持的情况下，如何安全地写回 ID 寄存器的值。

总体来看，本周的讨论集中在对补丁的审查、建议和潜在问题的解决方案上，显示出社区对补丁系列的积极反馈和对细节的关注。
