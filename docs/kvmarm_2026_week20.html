[PATCH v14 00/44] arm64: Support for Arm CCA in KVM

47 messages 2 participants Started: 2026-05-13
本邮件线程讨论了针对 ARM64 架构下 KVM 的一系列补丁，主要集中在支持 Arm CCA（Confidential Compute Architecture）和 RMM（Realm Management Monitor）相关功能的实现。

1. **原始补丁内容**：
本系列补丁（[PATCH v14 00/44]）旨在为 KVM 添加对 Arm CCA 的支持，允许在保护虚拟机中运行。补丁基于 v7.1-rc1，目标是 RMM v2.0-bet1。

2. **历史讨论要点**：
之前的讨论主要集中在如何实现 RMM 的功能，包括状态管理、内存跟踪、以及如何处理虚拟机的创建和销毁等。补丁中提到的主要更新包括对 RMM 的 API 进行调整，简化用户 API，支持状态 RMI 操作（SRO），以及改进中断和内存管理的处理。

3. **本周的新讨论与进展**：
- 本周的讨论中，补丁逐步完善了对 RMM 功能的支持，包括对内存的动态分配、RIPAS（Realm Initial Physical Address Space）管理、以及对 VGIC（虚拟通用中断控制器）的支持。
- 讨论还涉及了如何处理 Realm 中的 MMIO（内存映射 I/O）操作，确保在 Realm 中的虚拟机能够正确处理 I/O 请求。
- 另外，补丁中引入了对 SVE（Scalable Vector Extension）支持的检查，以确保虚拟机能够正确处理 SVE 寄存器。
- 最后，补丁还增加了对用户空间的支持，使其能够通过 KVM 接口配置和管理 Realm 的状态。

综上所述，本周的讨论和补丁更新在实现对 Arm CCA 的支持方面取得了显著进展，涵盖了内存管理、虚拟机状态管理和中断处理等多个方面。
